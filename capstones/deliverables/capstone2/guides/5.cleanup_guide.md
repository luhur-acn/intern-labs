# Cleanup & Optimization 🧹

Setelah menyelesaikan seluruh fase, pastikan semua sumber daya AWS yang dibuat telah dihapus agar tidak menarik biaya yang tidak perlu.

## 5.1 Hapus Resource yang Dikelola Terragrunt

Jalankan perintah berikut di dalam folder `infrastructure/` agar Terragrunt menghancurkan semua resource sesuai urutan graf ketergantungan (EC2 -> SG -> VPC -> S3).

```bash
terragrunt run-all destroy --terragrunt-non-interactive
```

---

## 5.2 Hapus State Backend (Manual)

Karena S3 Bucket dan DynamoDB untuk state dibuat secara manual, Anda harus menghapusnya juga secara manual via AWS CLI.

### 1. Hapus Tabel DynamoDB

```bash
aws dynamodb delete-table --table-name iac-capstone-tfstate-lock
```

### 2. Bersihkan dan Hapus S3 Bucket

```bash
# Kosongkan bucket dulu
aws s3 rm s3://iac-capstone-tfstate-[yourname] --recursive

# Hapus bucket
aws s3 rb s3://iac-capstone-tfstate-[yourname]
```
