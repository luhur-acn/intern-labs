## 3.2 Refactor dengan `state mv`

Simulasikan perubahan nama resource internal di modul tanpa menghancurkan resource asli di AWS.

1. Ubah nama resource di `main.tf` modul EC2.
2. Jalankan `terragrunt state mv` untuk memindahkan metadata state ke nama baru.
3. Verifikasi dengan `terragrunt plan` bahwa tidak ada resource yang akan dihancurkan/dibuat ulang.

---

## 3.3 Recovery Lock State

Simulasikan kebocoran proses (stuck lock) dan pulihkan dengan `force-unlock`.

- Hentikan paksa perintah `apply`.
- Ambil Lock ID dari pesan error.
- Jalankan `terragrunt force-unlock [ID]`.

---

## 3.4 Disaster Recovery (Export & Import)

Simulasikan skenario di mana state hilang atau rusak.

1. Backup state: `terragrunt state pull > backup.tfstate`.
2. Hapus state (Dorong file kosong): `terragrunt state push /dev/stdin` (dengan JSON kosong).
3. Verifikasi plan ingin membuat ulang semuanya.
4. Pulihkan state dari backup.
