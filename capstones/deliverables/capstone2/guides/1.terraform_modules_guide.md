# Phase 1: Author Terraform Modules 📦

Pada fase ini, Anda akan membangun modul Terraform yang reusable dari awal. **Aturan Utama**: Tidak diperbolehkan menggunakan modul komunitas (Terraform Registry). Semua harus ditulis sendiri.

## Struktur Direktori Modul

```text
modules/
├── vpc/
├── security-groups/
├── ec2/
└── s3/
```

Setiap modul harus memiliki minimal tiga file: `main.tf`, `variables.tf`, dan `outputs.tf`.

---

## 1.1 Modul VPC (`modules/vpc/`)

Modul ini mengelola infrastruktur jaringan dasar.

**Resources yang dikelola:**

- VPC
- Public & Private Subnets (menggunakan `for_each`)
- Internet Gateway & NAT Gateway
- Route Tables & Associations

**Pola `for_each` untuk Subnet:**
Gunakan `zipmap` untuk menggabungkan Availability Zones dengan CIDR blocks sehingga jumlah subnet fleksibel berdasarkan input.

---

## 1.2 Modul Security Groups (`modules/security-groups/`)

Menangani aturan firewall dengan pendekatan dinamis.

**Resources yang dikelola:**

- Security Groups (menggunakan `for_each`)
- Ingress & Egress Rules sebagai resource terpisah (bukan inline).

**Catatan Desain**: Gunakan resource `aws_vpc_security_group_ingress_rule` agar tidak terjadi konflik state saat menambahkan aturan baru.

---

## 1.3 Modul EC2 (`modules/ec2/`)

Meluncurkan instance komputasi dengan keamanan minimalis.

**Resources yang dikelola:**

- EC2 Instances (`for_each`)
- Shared IAM Role & Instance Profile
- SSM Policy Attachment (agar bisa login tanpa SSH Key).

---

## 1.4 Modul S3 (`modules/s3/`)

Mengelola storage objek dengan standar keamanan tinggi.

**Resources yang dikelola:**

- S3 Buckets (`for_each`)
- Block Public Access (Sangat Penting!)
- Server Side Encryption & Versioning
- Lifecycle Rules & Random ID Suffix untuk nama unik global.

---

## 1.5 Validasi Modul (Lokal)

Sebelum masuk ke Terragrunt, pastikan setiap modul dapat berjalan secara mandiri.

```bash
cd modules/[nama-modul]
terraform init
terraform validate
```

---

**Deliverables Checklist:**

- [ ] Kode modul di `modules/vpc/`, `security-groups/`, `ec2/`, dan `s3/`.
- [ ] Bukti `terraform validate` berhasil untuk semua modul.

- [ ] Bukti `terraform plan` berhasil menggunakan input yang skalabel (misal: 3 AZ).
