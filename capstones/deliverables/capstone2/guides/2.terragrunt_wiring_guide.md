# Phase 2: Terragrunt Multi-Environment Wiring 🌐

Pada fase ini, Anda akan menyambungkan modul-modul yang telah dibuat menggunakan Terragrunt. Terragrunt membantu menjaga kode agar tetap DRY (Don't Repeat Yourself).

## Struktur Direktori Infrastructure

```text
infrastructure/
├── root.hcl               # Global Backend & Provider
├── dev/                   # Lingkungan Development
│   ├── env.hcl            # Variabel Dasar (Dev)
│   ├── vpc/               # Stack VPC (Dev)
│   ├── security-groups/   # Stack SG (Dev)
│   ├── ec2/               # Stack EC2 (Dev)
│   └── s3/                # Stack S3 (Dev)
└── prod/                  # Lingkungan Production
    ├── env.hcl            # Variabel Dasar (Prod)
    ├── vpc/               # Stack VPC (Prod)
    ├── security-groups/   # Stack SG (Prod)
    ├── ec2/               # Stack EC2 (Prod)
    └── s3/                # Stack S3 (Prod)
```

---

## 2.1 Konfigurasi Global (`root.hcl`)

Satu file untuk mengelola backend S3/DynamoDB dan provider AWS untuk semua stack.

---

## 2.2 Wire Lingkungan DEV (`infrastructure/dev/`)

- Gunakan `dependency` blocks untuk meneruskan output antar stack.
- Contoh: `ec2` memerlukan `vpc_id` dan `private_subnet_ids` dari stack `vpc`.
- Gunakan `mock_outputs` agar perintah `plan` tetap berjalan meskipun resource belum ada.

---

## 2.3 Wire Lingkungan PROD (`infrastructure/prod/`)

Replikasikan wiring dari `dev` ke `prod` secara independen.

- `prod` harus menggunakan CIDR `10.1.0.0/16`.
- Tambahkan resource lebih banyak untuk `prod` (misal: 3 AZ, lebih banyak instance/bucket).

---

## 2.4 Deploy Seluruh Infrastruktur

Jalankan perintah berikut dari folder `infrastructure/` untuk mendeploy semua lingkungan sekaligus.

```bash
terragrunt run-all apply --terragrunt-non-interactive
```

Terragrunt akan membangun graf ketergantungan dan mendeploy stack dengan urutan yang benar (VPC -> SG -> EC2 -> S3).
