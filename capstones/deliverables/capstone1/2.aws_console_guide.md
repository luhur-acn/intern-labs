# Panduan Konfigurasi Infrastruktur AWS Console: Capstone Project

Panduan ini berisi langkah-langkah _step-by-step_ untuk mengonfigurasi infrastruktur cloud sesuai dengan dokumen desain arsitektur menggunakan AWS Management Console.

---

## 1. Jaringan (VPC & Networking) 🌐

Opsi **VPC and more** memungkinkan kita membuat seluruh topologi jaringan (VPC, Subnet, IGW, NAT GW, dan Endpoint S3) secara sekaligus dalam satu alur kerja.

### A. Konfigurasi "VPC and more"

1. Masuk ke **VPC Dashboard** -> Klik **Create VPC**.
   ![alt text](asset/image.png)
2. Pilih opsi **VPC and more**.
3. **Name tag auto-generation**: Isi `Capstone`.
4. **IPv4 CIDR block**: `10.0.0.0/16`.
   ![alt text](asset/image-1.png)
5. **Number of Availability Zones (AZs)**: Pilih `2`.
6. **Number of Public subnets**: Pilih `2`.
7. **Number of Private subnets**: Pilih `2`.
   ![alt text](asset/image-2.png)
8. **NAT Gateways ($)**: Pilih `In 1 AZ` (Sesuai desain untuk optimasi biaya).
9. **VPC Endpoints**: Pilih `S3 Gateway`.
10. Tinjau diagram visual di sebelah kanan untuk memastikan:
    - 2 Public Subnets & 2 Private Subnets tersebar di 2 AZ.
    - NAT Gateway terpasang di salah satu Public Subnet.
    - S3 Gateway terhubung ke Route Table.
11. Klik **Create VPC & resources**.
    ![alt text](asset/image-3.png)
    ![alt text](asset/image-4.png)

### B. Aktivasi Auto-Assign IP (Public Subnets)

Sangat penting: "VPC and more" tidak otomatis mengaktifkan public IP untuk instance di subnet publik.
![alt text](asset/image-5.png)

1. Pilih **Subnets** di menu kiri.
2. Cari dan pilih `Capstone-subnet-public1-us-east-1a`.
3. Klik **Actions** -> **Edit subnet settings**.
4. Centang **Enable auto-assign public IPv4 address**.
   ![alt text](asset/image-6.png)
5. Klik **Save**.
6. Ulangi langkah ini untuk `Capstone-subnet-public2-us-east-1b`.

## 4. Compute & Scaling (ALB & ASG) 🚀

### A. Membuat Target Group

1. Di VPC Dashboard, pilih **Target Groups**.
2. Klik **Create target group**.
3. **Target type**: `Instances`.
   ![alt text](asset/image-19.png)
4. **Name**: `Capstone-TG`.
5. **Protocol**: `HTTP`, **Port**: `80`, **VPC**: `Capstone-VPC`.
6. Klik **Next** lalu **Create target group** (tanpa mendaftarkan instance secara manual).
   ![alt text](asset/image-20.png)

### B. Membuat Application Load Balancer (ALB)

1. Pilih **Load Balancers** -> **Create load balancer**.
2. Pilih **Application Load Balancer**.
3. **Name**: `Capstone-ALB`.
   ![alt text](asset/image-21.png)
4. **Network mapping**: Pilih `Capstone-VPC` dan centang kedua **Public Subnets**.
   ![alt text](asset/image-22.png)
   ![alt text](asset/image-23.png)
5. **Security groups**: Pilih `ALB-SG`.
   ![alt text](asset/image-24.png)
6. **Listeners and routing**: HTTP:80 forward to `Capstone-TG`.
   ![alt text](asset/image-25.png)
7. Klik **Create load balancer**.
   ![alt text](asset/image-26.png)

### C. Membuat Launch Template

1. Masuk ke **EC2 Console** -> **Launch Templates**.
2. Klik **Create launch template**.
3. **Name**: `Capstone-LT`.
   ![alt text](asset/image-27.png)
4. **AMI**: `Amazon Linux 2023 AMI (ARM64)`.
   ![alt text](asset/image-28.png)
5. **Instance type**: `t4g.nano`.
   ![alt text](asset/image-29.png)
6. **IAM instance profile**: `CapstoneEC2Role`.
   ![alt text](asset/image-30.png)
7. **Security Groups**: `EC2-SG`.
   ![alt text](asset/image-31.png)
8. **Advanced details** -> **User data**: Masukkan script berikut (sudah mendukung IMDSv2 untuk Amazon Linux 2023):

   ```bash
   #!/bin/bash
   yum update -y
   yum install -y httpd stress

   # Mendapatkan Token IMDSv2
   TOKEN=$(curl -X PUT "http://169.254.169.254/latest/api/token" -H "X-aws-ec2-metadata-token-ttl-seconds: 21600")

   # Mengambil metadata menggunakan token
   INSTANCE_ID=$(curl -H "X-aws-ec2-metadata-token: $TOKEN" -s http://169.254.169.254/latest/meta-data/instance-id)
   AZ=$(curl -H "X-aws-ec2-metadata-token: $TOKEN" -s http://169.254.169.254/latest/meta-data/placement/availability-zone)

   # Membuat file HTML
   cat > /var/www/html/index.html <<EOF
   <html>
   <head>
       <title>Capstone Web App</title>
       <style>
           body { font-family: sans-serif; text-align: center; padding-top: 50px; background-color: #f4f4f9; }
           .card { border: 1px solid #ddd; padding: 20px; display: inline-block; border-radius: 10px; background: white; box-shadow: 0 4px 8px rgba(0,0,0,0.1); }
           h1 { color: #232f3e; }
       </style>
   </head>
   <body>
       <div class="card">
           <h1>Capstone Architecture — Production Web App</h1>
           <p><strong>Instance ID:</strong> $INSTANCE_ID</p>
           <p><strong>Availability Zone:</strong> $AZ</p>
       </div>
   </body>
   </html>
   EOF

   # Health check endpoint
   mkdir -p /var/www/html/health
   echo "OK" > /var/www/html/health/index.html

   systemctl start httpd
   systemctl enable httpd
   ```

   ![alt text](asset/image-32.png)

### D. Membuat Auto Scaling Group (ASG)

1. Pilih **Auto Scaling Groups** -> **Create Auto Scaling group**.
2. **Name**: `Capstone-ASG`.
   ![alt text](asset/image-33.png)
3. **Launch template**: `Capstone-LT`.
4. **VPC**: `Capstone-VPC`,
   ![alt text](asset/image-34.png)
   **Subnets**: Pilih kedua **Private Subnets**.
5. **Load balancing**: Attach to an existing load balancer -> `Capstone-TG`.
   ![alt text](asset/image-35.png)
6. **Health checks**:
   - Centang **Enable ELB health checks**.
   - Set **Health check grace period** to `300 seconds`.
   - _Justifikasi: Dengan EC2 health checks, ASG hanya mengganti instance jika VM mati. Dengan ELB health checks, ASG juga akan mengganti instance jika web server di dalamnya crash (gagal health check ALB), menjamin ketersediaan aplikasi yang lebih baik._
     ![alt text](asset/image-36.png)
7. **Group size**:
   - Desired: `2`
   - Min: `2`
   - Max: `4`
     ![alt text](asset/image-37.png)
8. **Scaling policies**: Pilih **Custom scaling policies** untuk menggunakan **Step Scaling** (Manual >70% dan <30%).
   - _Detail langkah konfigurasi manual (Alarm & Policy) dapat dilihat di Bagian 5 di bawah._

9. Klik **Create Auto Scaling group**.
   ![alt text](asset/image-39.png)

## 5. Konfigurasi Scaling pada ASG yang Sudah Ada 🔄

Jika ASG Anda sudah terlanjur dibuat tanpa kebijakan scaling, Anda tetap bisa menambahkannya melalui tab **Automatic scaling**.

### A. Cara Menambahkan Policy

1. Masuk ke **EC2 Console** -> **Auto Scaling Groups**.
2. Pilih ASG yang ingin dikonfigurasi (`Capstone-ASG`).
3. Klik tab **Automatic scaling** di bagian bawah.
4. Di bagian **Dynamic scaling policies**, klik **Create dynamic scaling policy**.

### B. Manual Step Scaling: Workflow Alarm -> Policy

Sesuai arsitektur Anda, berikut adalah langkah manual untuk mengatur scaling diskrit (>70% tambah 1, <30% kurang 1):

#### Tahap 1: Buat Alarms di CloudWatch

1. Masuk ke **CloudWatch Console** -> **Alarms** -> **Create alarm**.
2. **Metric**: Pilih `EC2` -> `By Auto Scaling Group` -> Cari `CPUUtilization` untuk ASG Anda.
3. **Conditions**:
   - **Scale-Out Alarm**: Threshold > `70`, Type: `Static`.
   - **Scale-In Alarm**: Threshold < `30`, Type: `Static`.
4. Pilih periode (misal 1 menit) dan jumlah periode eval (misal 2 consecutive periods).
5. Beri nama (misal: `Alarm-CPU-High-70` dan `Alarm-CPU-Low-30`).
   ![alt text](asset/image-45.png)

#### Tahap 2: Buat Scaling Policies di ASG

1. Kembali ke **EC2 Dashboard** -> **Auto Scaling Groups** -> Tab **Automatic scaling**.
2. Klik **Create dynamic scaling policy**.
3. **Policy type**: Pilih `Step scaling`.
4. **Policy name**: `Scale-Out-Policy`.
5. **CloudWatch alarm**: Pilih alarm `Alarm-CPU-High-70` yang tadi dibuat.
6. **Action**: `Add 1 capacity units`.
7. Klik **Create**.
8. Ulangi langkah di atas untuk **Scale-In-Policy**:
   - Pilih alarm `Alarm-CPU-Low-30`.
   - **Action**: `Remove 1 capacity units`.
     ![alt text](asset/image-46.png)

## 6. Observabilitas (CloudWatch) 📈

Selain alarm untuk scaling, Anda sebaiknya memantau kesehatan aplikasi:

### A. Alarm 5xx Errors (ALB)

1. Buka **CloudWatch Console** -> **Alarms** -> **Create alarm**.
2. Metric: `ApplicationELB` -> `Per AppELB, Per TG Metrics` -> `HTTPCode_ELB_5XX_Count`.
3. Threshold: `> 5` selama `1` periode.
4. Notification: Konfigurasi SNS Topic jika ingin menerima email alert jika aplikasi bermasalah.
   ![alt text](asset/image-47.png)
